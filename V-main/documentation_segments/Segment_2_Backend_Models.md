# Segment 2: Backend Core - Data Models & Domain Entities

This segment contains the full source code for the core domain models used in Project **ILLUSION**. These entities define the database schema and the relationships between assessments, candidates, and results.

## 1. Assessment Entity (backend/src/main/java/com/proctoring/model/Assessment.java)
The `Assessment` entity is the top-level container for any test created by an administrator. It manages the metadata and the list of associated questions.

```java
package com.proctoring.model;

import jakarta.persistence.*;
import lombok.*;
import java.time.LocalDateTime;
import java.util.List;

@Entity
@Data
@NoArgsConstructor
@AllArgsConstructor
@Builder
public class Assessment {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;
    @Column(columnDefinition = "TEXT")
    private String description;
    
    private Integer durationMinutes;
    private String status; // ACTIVE, INACTIVE, DRAFT
    
    private boolean clarityCheckEnabled; // AI Voice Interview toggle
    
    @OneToMany(mappedBy = "assessment", cascade = CascadeType.ALL)
    private List<QuizQuestion> quizQuestions;

    @OneToMany(mappedBy = "assessment", cascade = CascadeType.ALL)
    private List<CodingQuestion> codingQuestions;

    private LocalDateTime createdAt;
    
    @PrePersist
    protected void onCreate() {
        createdAt = LocalDateTime.now();
    }
}
```

## 2. Exam Session Entity (backend/src/main/java/com/proctoring/model/ExamSession.java)
This entity tracks the lifecycle of a specific candidate taking a specific test. It is used to generate the temporary JWT tokens for secure exam access.

```java
@Entity
@Data
public class ExamSession {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @ManyToOne
    private Assessment assessment;

    @ManyToOne
    private Candidate candidate;

    private String testToken; // Unique hash for the test URL
    private String status; // START, IN_PROGRESS, COMPLETED, EXPIRED
    
    private LocalDateTime startTime;
    private LocalDateTime endTime;
    private String candidateIp;
    
    private Integer violationCount = 0;
}
```

## 3. The Result & Penalty Entity (backend/src/main/java/com/proctoring/model/Result.java)
The `Result` entity persists the final calculated scores. It includes logic for weighting different sections (Coding vs MCQ) and applying the AI-calculated proctoring penalty.

```java
@Entity
@Data
public class Result {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private Long sessionId;
    private String candidateName;
    private String candidateEmail;

    // Section Scores
    private Double quizScore;
    private Double codingScore;
    private Double oralScore;
    private Double finalScore; // The normalized total

    private Integer violationCount;
    private Double penaltyApplied; // Expressed as a percentage (e.g., 0.95)
    
    private String verdict; // SELECTED, REVIEW, REJECTED
    private String aiSummary; // Generated by Gemini based on proctoring logs
    
    private LocalDateTime processedAt;
}
```
